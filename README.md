## 3. Описание проекта (предметная область) ##
Название проекта: OrangeCheck
Предметная область проекта — система учёта посещаемости студентов в образовательных учреждениях. В учебных заведениях преподаватели ежедневно фиксируют присутствие или отсутствие студентов на занятиях. Традиционно это делается в бумажных журналах или таблицах, что может приводить к ошибкам, потере данных и неудобству при анализе посещаемости.
Разрабатываемое веб-приложение предназначено для автоматизации процесса учёта посещаемости студентов. Система позволит преподавателям быстро отмечать присутствие студентов на занятиях, хранить данные в электронной базе и формировать отчёты.
Основные пользователи системы:
•	Преподаватели — отмечают посещаемость студентов на занятиях.
•	Администраторы — управляют списками студентов, групп и преподавателей.
•	Студенты (опционально) — могут просматривать свою посещаемость.
Приложение будет работать через веб-интерфейс, что позволит использовать его с любого устройства, имеющего доступ к интернету. Все данные будут храниться в базе данных и доступны для просмотра и анализа.

Описание:
Возвращает список всех студентов, зарегистрированных в системе.

      "id": 2,
      "name": "Анна Петрова",
      "group": "ИС-21",
      "email": "petrova@example.com"
    }
  ]
}

POST /students
Добавление нового студента.
Описание:

Пример запроса:
{
  "name": "Алексей Смирнов",
  "group": "ИС-22",
  "email": "smirnov@example.com"
}

GET /groups
Получение списка учебных групп.

    }
  ]
}

POST /attendance
Добавление записи о посещаемости.

GET /attendance

{

## 1. Краткое описание реализованной серверной части
Серверная часть приложения разработана на языке Python с использованием современного веб-фреймворка FastAPI. Основная задача сервера — обработка запросов от клиентской части, работа с данными студентов и фиксация их присутствия в журнале. Выбранная архитектура позволяет легко расширять функционал системы (например, добавлять новые группы или предметы).
## 2. Структура серверной части
Для обеспечения чистоты кода проект разделен на модули.
Дерево файлов:
•	app — корневая папка сервера.
•	mainn.py — точка входа, содержит описание путей (API) и логику обработки запросов.
•	models.py — содержит описание таблиц базы данных.
•	database.py — отвечает за подключение к базе данных SQLite.
•	schemas.py — описывает формат данных, которые сервер принимает и отправляет. 
Схема взаимодействия:
Пользователь (Браузер) <--> Маршруты (main.py) <--> Валидация (schemas.py) <--> База данных (models.py)
## 3. Основные функции сервера
•	get_students(group: str):
•	Описание: Получает список студентов для конкретной группы.
•	Входные параметры: group (строка, например, "ИС-21").
•	Возвращаемое значение: Список объектов (ID, ФИО, текущий статус).
•	save_attendance(data: list):
•	Описание: Сохраняет измененные данные о посещаемости в базу данных.
•	Входные параметры: Массив данных со статусами студентов.
•	Возвращаемое значение: Сообщение об успешном выполнении (success).

## 1. Краткое описание клиентской части
Клиентская часть (Frontend) представляет собой интерактивную веб-страницу, реализованную с помощью технологий HTML5, CSS3 и JavaScript. ## 2. Основные блоки страницы и их назначение

•	Подвал (Footer): Содержит функциональные кнопки для массового действия («Отметить всех») и сохранения итогового результата.
## 3. Схема навигации и работа элементов
Приложение реализовано по принципу SPA (Single Page Application). Вся работа происходит на одной странице, что исключает лишние перезагрузки.
•	Переключатели статуса: Реализованы на базе HTML-элементов radio, что гарантирует выбор только одного из двух состояний для каждого студента.
•	Кнопка «Отметить всех "Присутствует"»: Автоматизирует рутинный процесс, проставляя положительный статус всем студентам в списке одним кликом.
•	Кнопка «Сохранить»: Инициирует сбор данных со всех полей таблицы и имитирует их отправку в базу данных.
## 4. Запросы клиента к серверу
Взаимодействие с бэкендом происходит посредством JSON-запросов:
•	Пример запроса на сохранение (POST):
URL: /api/attendance
Body: [{"id": 2, "name": "Анна Петрова", "status": "absent"}]

2.	Настройка: На панели фильтров проверяется соответствие группы и предмета.
3.	Первичная отметка: Нажимается кнопка «Отметить всех "Присутствует"», чтобы быстро заполнить таблицу.

5.	Финализация: Нажимается кнопка «Сохранить». Система выдает уведомление об успешной записи данных.
