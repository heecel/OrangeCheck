3. Описание проекта (предметная область)
Название проекта: OrangeCheck
Предметная область проекта — система учёта посещаемости студентов в образовательных учреждениях. В учебных заведениях преподаватели ежедневно фиксируют присутствие или отсутствие студентов на занятиях. Традиционно это делается в бумажных журналах или таблицах, что может приводить к ошибкам, потере данных и неудобству при анализе посещаемости.
Разрабатываемое веб-приложение предназначено для автоматизации процесса учёта посещаемости студентов. Система позволит преподавателям быстро отмечать присутствие студентов на занятиях, хранить данные в электронной базе и формировать отчёты.
Основные пользователи системы:
•	Преподаватели — отмечают посещаемость студентов на занятиях.
•	Администраторы — управляют списками студентов, групп и преподавателей.
•	Студенты (опционально) — могут просматривать свою посещаемость.
Приложение будет работать через веб-интерфейс, что позволит использовать его с любого устройства, имеющего доступ к интернету. Все данные будут храниться в базе данных и доступны для просмотра и анализа.

5. Макеты интерфейса
Интерфейс веб-приложения должен быть простым и понятным для пользователей. Основные страницы системы представлены следующими макетами.
Страница посещаемости
На данной странице преподаватель отмечает посещаемость студентов.
Элементы страницы:
•	выбор группы;
•	выбор даты занятия;
•	список студентов;
•	отметка «присутствует / отсутствовует»;
•	кнопка «Сохранить»;
•	кнопка «Отметить всех «Присутствует».
 

Страница авторизации
Предназначена для входа пользователей в систему.
Элементы страницы:
•	поле логин;
•	поле пароль;
•	кнопка «Войти»;
•	сообщение об ошибке при неправильном вводе данных.
Пользователь вводит свои данные и получает доступ к системе в зависимости от своей роли (администратор или преподаватель).
Главная страница (панель управления)
После входа пользователь попадает на главную страницу системы.
Элементы страницы:
•	меню навигации;
•	список доступных разделов;
•	краткая информация о системе.
Основные разделы меню:
•	Студенты
•	Группы
•	Посещаемость
•	Отчёты
 Страница списка студентов
На этой странице отображается список всех студентов.
Элементы страницы:
•	таблица студентов;
•	кнопка «Добавить студента»;
•	кнопка «Редактировать»;
•	кнопка «Удалить»;
•	поле поиска студентов.
В таблице отображаются:
•	имя студента;
•	группа;
•	номер студента.
Страница отчётов
Предназначена для просмотра статистики посещаемости.
Элементы страницы:
•	выбор группы;
•	выбор периода времени;
•	таблица посещаемости;
•	кнопка «Сформировать отчёт»;
•	возможность скачивания отчёта.
6. Описание API
Формат данных: JSON
Аутентификация: JWT (для защищённых эндпоинтов)
GET /students
Получение списка студентов.
Описание:
Возвращает список всех студентов, зарегистрированных в системе.
Пример ответа:
{
  "data": [
    {
      "id": 1,
      "name": "Иван Иванов",
      "group": "ИС-21",
      "email": "ivanov@example.com"
    },
    {
      "id": 2,
      "name": "Анна Петрова",
      "group": "ИС-21",
      "email": "petrova@example.com"
    }
  ]
}

POST /students
Добавление нового студента.
Описание:
Создаёт нового студента в системе.
Пример запроса:
{
  "name": "Алексей Смирнов",
  "group": "ИС-22",
  "email": "smirnov@example.com"
}

GET /groups
Получение списка учебных групп.
Описание:
Возвращает список всех групп.
Пример ответа:
{
  "data": [
    {
      "id": 1,
      "name": "ИС-21"
    },
    {
      "id": 2,
      "name": "ИС-22"
    }
  ]
}

POST /attendance
Добавление записи о посещаемости.
Описание:
Преподаватель отмечает присутствие или отсутствие студента на занятии.
Пример запроса:
{
  "student_id": 1,
  "date": "2026-03-12",
  "status": "present"
}

GET /attendance
Получение информации о посещаемости.
Описание:
Возвращает данные о посещаемости студентов.
Пример ответа:
{
  "data": [
    {
      "student_id": 1,
      "date": "2026-03-12",
      "status": "present"
    },
    {
      "student_id": 2,
      "date": "2026-03-12",
      "status": "absent"
    }
  ]
}


7. Модель данных
Для хранения информации в системе используется база данных. Основные сущности системы: пользователи, студенты, группы и посещаемость.
Таблица Users (Пользователи)

Поле	Тип данных	Описание
id	INTEGER	Уникальный идентификатор пользователя
login	TEXT	Логин пользователя
password	TEXT	Пароль пользователя
role	TEXT	Роль пользователя (администратор, преподаватель)

Таблица Groups (Группы)

Поле	Тип данных	Описание
id	INTEGER	Уникальный идентификатор группы
name	TEXT	Название группы

Таблица Students (Студенты)

Поле	Тип данных	Описание
id	INTEGER	Уникальный идентификатор студента
name	TEXT	Имя и фамилия студента
group_id	INTEGER	Идентификатор группы
email	TEXT	Электронная почта студента

Таблица Attendance (Посещаемость)

Поле	Тип данных	Описание
id	INTEGER	Уникальный идентификатор записи
student_id	INTEGER	Идентификатор студента
date	DATE	Дата занятия
status	TEXT	Статус посещения (присутствовал / отсутствовал)

Связи между таблицами:

•	Один группа может содержать много студентов.
•	Один студент может иметь много записей о посещаемости.
•	Таблица посещаемости связана со студентами через поле student_id.
Индексы для производительности
Для ускорения работы с базой данных можно добавить несколько индексов:

•	Attendance(student_id) — чтобы быстро получать все записи о посещаемости конкретного студента.
•	Students(group_id) — чтобы быстро получать список студентов в группе.
•	Groups(name) — для быстрого поиска группы по названию.
